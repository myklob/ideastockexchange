# PWA Icons

`icon-192.png` and `icon-512.png` are referenced by `/manifest.webmanifest`
and consumed by Android Chrome's "Install app" flow plus iOS's
`apple-touch-icon` fallback.

The current files are **placeholder solid-color squares** generated by
`tools/generate-icons.mjs`. They satisfy the manifest schema and let the
install prompt fire, but they are not real artwork.

## Replacing the placeholders

Drop in real PNGs at the same paths and same dimensions. Both sizes must
exist; the manifest declares `purpose: "any maskable"` so the icons should
keep their important visual content inside the inner 80% safe zone (Android
applies an adaptive mask that crops the outer 10–20%).

If you want to regenerate the placeholders later (different color, etc.),
edit the `COLOR` constant in `tools/generate-icons.mjs` and re-run:

```bash
node tools/generate-icons.mjs
```
