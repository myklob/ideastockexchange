🔗 Linkage Arguments

Each argument below tests the bridge between X and Y, not the truth of X or Y. Reasons to agree explain why the connection is direct, necessary, or unavoidable. Reasons to disagree explain why the connection breaks: missing steps, mismatched scope, alternative explanations, or the True But Irrelevant pattern.

✅ Reasons the linkage is strong ❌ Reasons the linkage is weak
1. [Mechanism: X causes Y through a named pathway] [1-3 sentences naming the causal or logical mechanism. The bridge from X to Y is direct, with few intermediate steps. Branch out to the relevant canonical page if the mechanism has its own treatment.] 1. [Missing intermediate step] [1-3 sentences identifying what hidden assumption must be true for X to support Y. Link to Assumptions if the gap reveals a foundational premise.]
2. [Necessity: if X is true, Y must be strengthened] [Explain why no plausible scenario allows X to be true while Y stays unchanged.] 2. [True But Irrelevant: X is true but does not bear on Y] [Identify why X, even granted as fact, does not move the needle on Y. This is the most common linkage failure mode.]
3. [Scope fit: X's domain matches Y's domain] [Population, timeframe, and context align between X and Y.] 3. [Scope or scale mismatch] [X is from a different population, scale, or time period than Y. Extrapolation fails.]
4. [Monotonicity: more X means more Y, consistently] [The relationship doesn't reverse direction at any plausible value of X.] 4. [Parent-mechanism mismatch] [X belongs under a different parent claim than Y. The fluent reading is misleading. (See Schiltz case in Failure Mode Worked Examples below.)]

Linkage arguments are scored by the same recursive engine as belief arguments. Each row above has its own argument score; the linkage score for this page is the net aggregate. See Argument Scores from Sub-Argument Scores.

📖 Definitions and Scoring Concepts

Linkage Score: How directly X strengthens or weakens Y. Range is −1.0 (refutation) to +1.0 (proof), with 0 meaning irrelevant. Computed as Relevance × Network Strength × Contextual Fit × Uniqueness. See Linkage Scores.

ECLS vs ACLS: ECLS is Evidence-to-Conclusion Linkage (a study supports a conclusion). ACLS is Argument-to-Conclusion Linkage (a logical claim supports a conclusion). The five-step check applies to both.

Necessity / Sufficiency / Directness: The three diagnostic questions used to derive the score. Necessity asks whether Y suffers if X is false; Sufficiency asks whether X alone justifies Y; Directness counts the logical steps from X to Y.

Contribution to parent: An argument's effect on its parent claim equals Argument Score × Linkage Score. A perfect argument with a 0.3 linkage contributes less than a moderate argument with a 0.9 linkage. See Argument scores from sub-argument scores.

Equivalency Score: How similar two phrasings of the same statement are. Used in the rephrasings table to test linkage robustness. See the Belief Equivalency Engine.

Audit-lock: Linkage scores are computed from sub-arguments, never assigned by hand. If the score on this page changes, the change traces to a specific edit in the linkage argument tree above.

Staleness: A linkage validated more than 365 days ago is flagged stale and the five-step check should be re-run. Stale linkages don't lose their score automatically; they get a yellow indicator until re-validated.
